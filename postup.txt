E-SHOP PROJEKT - D1 REPOSITORY PATTERN
=======================================

SQL (src/sql/)
--------------
01_create_tables.sql - 5 tabulek (users, categories, products, orders, order_items)
02_create_views.sql  - 2 views (v_order_details, v_product_stats)
03_seed_data.sql     - testovaci data

BACKEND (src/backend/)
----------------------
config.py    - konfigurace (DB, Flask)
database.py  - pripojeni k DB, execute_query, execute_non_query
app.py       - Flask API endpointy

repositories/
  base_repository.py     - zaklad (get_all, get_by_id, delete, count)
  product_repository.py  - CRUD produkty + get_featured, update_stock
  category_repository.py - CRUD kategorie
  user_repository.py     - CRUD users + transfer_credits (TRANSAKCE)
  order_repository.py    - create_order (CRUD 3 tabulky, TRANSAKCE)

FRONTEND (src/frontend/)
------------------------
app/
  page.js           - homepage (featured produkty)
  layout.js         - layout + navigace
  globals.css       - styly

  products/
    page.js         - seznam produktu
    [id]/page.js    - detail produktu

  cart/
    page.js         - kosik

  admin/
    page.js         - admin dashboard + report
    products/page.js - sprava produktu
    import/page.js   - import JSON

components/
  Navbar.js         - navigace
  ProductCard.js    - karta produktu
  CartItem.js       - polozka kosiku

lib/
  api.js            - API volani na backend

API ENDPOINTY
-------------
GET  /api/products          - vsechny produkty
POST /api/products          - novy produkt
GET  /api/orders/<id>       - detail objednavky
POST /api/orders            - nova objednavka (3 tabulky!)
POST /api/users/transfer    - prevod kreditu (transakce!)
GET  /api/report            - agregace 3+ tabulek
POST /api/import/products   - import JSON
